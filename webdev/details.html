Details for nerds

Let's walk through what's under the hood.

Details for basic websites

I host static websites on GitHub pages. Deployments to any necessary servers (e.g., GoDaddy) are often handled by Netlify. Because GitHub pages can't handle any server-side logic, I run forms (with reCAPTCHA!) through free Formspree plans — though, of course, upgrades are fair game! Google Docs integrations are performed with similarly open-source tools. HTML5 UP provides most of the templates that I work wit; I'm a huge fan of their templates because they're lightweight, respond really well to mobile, desktop, and even tablets, and are quite modular. In general, not much beyond HTML, CSS, and relatively vanilla JS are needed for this package.

Details for automated websites

Turns out, you can still do a lot of cool stuff with static websites! I run much of my automation through GitHub Actions. Daily cron jobs allow me to check that every link on a given site is working. Also, I can scrape Google Scholar or NASA/ADS with open-source Python tools.

Details for fully individualized websites

This is where things can get really exciting! MongoDB and React are a few tricks up my sleeve that I can bring into play for active websites (which I can generally host via Heroku). I'm also very interested in embedding IPython widgets into websites — especially to communicate science.
