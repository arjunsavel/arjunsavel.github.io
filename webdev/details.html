Details for nerds

Let's walk through what's under the hood.

Details for basic websites

I host static websites on GitHub pages. Doing so allows me to manage websites from GitHub repositories — basically, online places where code lives. Deployments to any necessary servers (e.g., GoDaddy) are often handled by Netlify, a (mostly) free service made for web apps. Because GitHub pages can't handle any server-side logic (i.e., it can't run any code when a user interacts with a website), I run forms (with reCAPTCHA to prevent spam!) through free Formspree plans — though, of course, upgrades are fair game! Google Docs integrations are performed with similarly open-source tools. HTML5 UP provides most of the templates that I work with; I'm a huge fan of their templates because they're lightweight; respond really well to mobile, desktop, and even tablets; and are quite modular. In general, not much beyond HTML, CSS, and relatively vanilla JS are needed for this package.

Tl;dr: I'll make you a clean, reliable website.

Details for automated websites

I run much of my automation through GitHub Actions, which is a free way to run automated code via GitHub repositories. For instance, daily cron jobs can check that every link on a given site is working. Also, I can scrape Google Scholar, NASA/ADS, or arXiv with open-source Python tools.

Tl;dr: I'll make you a website that automatically updates itself, getting better over time instead of stagnating.

Details for fully individualized websites

This is where things can get really exciting! MongoDB (a database tool) and React (a JavaScript library for user interface) are a few tricks up my sleeve that I can bring into play for active websites (which I can generally host via Heroku). I'm also very interested in embedding IPython widgets into websites — especially to communicate science.

Tl;dr: Talk to me. We'll make it happen!
